Azure page to configure Role

This window allows you to configure an Azure Role.

Step 1: Specify role name in ‘Role name’ text box. The role's name must be unique.

Step 2: Select VM Size using the drop down list. Set this value to change the size of the VM allotted to this role. The default value is Small.

Step 3: Enter the Instances element specifies the number of instances of this role to deploy as part of the service deployment. Default 1 will get selected.

Step 4: User can view endpoints in tabular format. Press Add button to add new endpint for specified role. Remove button will remove the endpoint from the specified role.

The Endpoints element describes the collection of input (external) and internal endpoints for a role. There are two types of endpoints :

· InputEndpoint

The InputEndpoint element describes an external endpoint for a worker role. You can define multiple endpoints that are a combination of HTTP, HTTPS, and TCP endpoints. You can specify any port number you choose for an input endpoint, but the port numbers specified for each role in the service must be unique. For example, if you specify that a role uses port 80 for HTTP and port 443 for HTTPS, you might then specify that a second role uses port 8080 for HTTP and port 8043 for HTTPS.

· InternalEndpoint

The InternalEndpoint element describes an internal endpoint to a worker role. An internal endpoint is available only to other role instances running within the service; it is not available to clients outside the service. A worker role may have up to five HTTP or TCP internal endpoints.

Step 5: You can enable remote debugging for this Azure role by checking the checkbox. A new endpoint is automatically created. You can change the port numbers or choose another endpoint. You can also specify that the JVM should be suspended until a debugger connection is established. Use the Create Debug Configurations button to automatically generate up to two new debug configurations: one for the emulator and one for the cloud.
